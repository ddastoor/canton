dpm.sh — What Each Step Does

Installer for the Digital Asset dpm SDK. Example values shown are for a Linux / x86_64 machine.

Step Artifact / Name Value Source
Fetch the latest published SDK version number VERSION 3.5.2 https://get.digitalasset.com/install/latest
Detect CPU architecture and normalize it (x86_64→amd64, aarch64→arm64) ARCH amd64 Local command: uname -m
Detect the operating system and lowercase it OS linux Local command: uname
Build the tarball filename from version, OS and arch TARBALL dpm-3.5.2-linux-amd64.tar.gz Computed locally (string built from VERSION, OS, ARCH)
Build the full download URL for that tarball TARBALL_URL https://get.digitalasset.com/install/dpm-sdk/dpm-3.5.2-linux-amd64.tar.gz https://get.digitalasset.com/install/dpm-sdk/
Create a temporary working directory TMPDIR /tmp/tmp.XXXXXXXXXX (random) Local command: mktemp -d
Download the tarball into the temp directory ${TMPDIR}/${TARBALL} /tmp/tmp.XXXX/dpm-3.5.2-linux-amd64.tar.gz https://get.digitalasset.com/install/dpm-sdk/dpm-3.5.2-linux-amd64.tar.gz
Create a sub-directory to extract into extracted /tmp/tmp.XXXX/extracted Local command: mkdir -p
Unpack the tarball (stripping the top-level folder) extracted files /tmp/tmp.XXXX/extracted/bin/dpm, … Local command: tar xzf … --strip-components 1
Run the downloaded binary to bootstrap/install dpm onto the system dpm install ${extracted}/bin/dpm bootstrap ${extracted} Local binary just extracted
Delete the temporary directory (cleanup) — rm -rf ${TMPDIR} Local command: rm -rf
